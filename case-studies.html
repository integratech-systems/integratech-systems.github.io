🏦 Banking NiFi + Kafka 3 weeks

SWIFT ETL Pipeline: From 3-Day Outages to 99.97% Uptime

A top-20 Russian bank had a SWIFT MT103 processing pipeline built on a brittle custom Java monolith. The system crashed under end-of-month load — every single month. The team was spending 2–3 days per incident on manual recovery.

We rebuilt the ingestion and transformation layer on Apache NiFi with Kafka as the durable backbone. Dead-letter handling, idempotent processors, and automated replay replaced the fragile manual procedures. Monitoring was wired to PagerDuty with runbooks for every alert.

  • Replaced monolithic SWIFT processor with NiFi flow + Kafka topic per message type
  • Implemented exactly-once semantics for financial message delivery
  • Automated reconciliation against НРД settlement records
  • Zero data loss during the first end-of-month cycle post-launch

📈 Trading WebSocket + PostgreSQL 6 weeks

МосБиржа Real-time Feed: 60% Latency Reduction on Market Data Pipeline

A proprietary trading firm was receiving МосБиржа market data through a third-party vendor feed. p99 latency was hitting 180ms during volatile sessions — unacceptable for their algorithmic strategies. The vendor refused to SLA-guarantee improvements.

We built a direct WebSocket connection to МосБиржа FAST/FIXS feed, designed a low-latency normalisation layer in Java 21 with virtual threads, and stored tick data directly into partitioned PostgreSQL tables with BRIN indexes. The p99 dropped from 180ms to 72ms on day one.

  • Direct FAST protocol decoder — no vendor intermediary
  • Java 21 virtual threads handling 12 000 concurrent instrument subscriptions
  • Partitioned PostgreSQL tick storage with sub-millisecond write latency
  • Grafana dashboard for latency percentiles, gap detection, and reconnects

🏛️ Government NiFi + Greenplum 4 months

ДИТ Москвы: City-Scale Data Integration for 40+ Source Systems

Moscow's Department of Information Technology needed a unified data integration layer to consolidate operational data from 40+ municipal systems into a Greenplum analytical platform. Existing ETL was a patchwork of scheduled SQL scripts with no lineage, no monitoring, and daily failures.

We designed and deployed an Apache NiFi cluster that handles the full ingestion lifecycle — from REST and SOAP source APIs to schema validation, transformation, error handling, and lineage tracking. Every pipeline self-documents its state through structured logging and a custom Grafana dashboard.

  • NiFi cluster: 3-node HA with ZooKeeper, 40+ active flows
  • Schema Registry for all source contracts — breaking changes caught before ingestion
  • Full data lineage from source API to Greenplum analytical table
  • On-call runbook and automated alerting handed to the internal team

🔐 Enterprise Headscale + OpenBao 2 weeks

Private Corporate Network: Zero Big-Tech Dependency for a 30-Person Team

A fintech company with remote employees across Russia, Georgia, and Israel needed a secure internal network that didn't route traffic through US-controlled cloud providers. Existing WireGuard setup required manual key management and had no access control policies.

We deployed a self-hosted headscale controller with Tailscale clients, implemented ACL policies for network segmentation, added an XMPP Prosody messaging server, and integrated OpenBao for secrets rotation. Client onboarding was reduced to scanning a QR code.

  • Headscale controller on private VPS, exit nodes in DE and US
  • ACL policies: per-user access tiers, internet exit control
  • Self-hosted Prosody XMPP with end-to-end encrypted group chats
  • QR-code based onboarding page — client setup under 2 minutes
